The iterative converter for vector types -- loops over the split-\/vector and converts the internal types. 

The {\ttfamily typename} argument allows the function to disable itself at compile time if T\+\_\+\+Inner is a char, thus inducing a Substitution Failure. Through SFINAE principles, this allows std\+::string (internally a std\+::vector$<$char$>$ in many ways) to be ignored by this, and thus picked up by the standard converter We allow vectors to be wrapped in either \mbox{[}\mbox{]}, \{\} or (). This function removes them for internal use.

\begin{DoxyWarning}{Warning}
We do {\itshape not} do any form of parsing or checking to allow nested braces. End caps are purely for user readability.
